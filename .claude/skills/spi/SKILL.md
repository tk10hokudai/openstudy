# SPI 試験 作問スキル

共通ルールは `.claude/skills/question-creator/SKILL.md` を先に参照すること。
このスキルはその試験固有の上書きルールと追加仕様を定める。

---

## 1. 試験概要

| 項目 | 内容 |
|---|---|
| 正式名称 | SPI 試験（exam_id=3） |
| seed ファイル | `supabase/seeds/spi.sql` |
| SQL パターン | パターン B（単純 INSERT + currval） |
| 問題形式 | **全問 4 択選択式**（`question_type='choice'`） |
| 選択肢シャッフル | 可能（正解を **1 番目** に記述） |

> **重要：非言語問題も選択式に統一する。**
> 旧 spi.sql には `question_type='text'`（数値入力）の非言語問題が存在するが、
> 問題を作りなおす際は **4 択選択式（`choice`）** で作成すること。

## 3. 出題分野と問題形式

### 3-1. 言語分野（section_id=12）

| カテゴリ | 内容 | 問題数目安 |
|---|---|---|
| 語句の意味 | 漢語・熟語・慣用表現の意味を4択で選ぶ | 30問 |
| 同意語 | 与えられた語と同じ意味の語を4択で選ぶ | 30問 |
| 反意語 | 与えられた語の反対の意味を持つ語を4択で選ぶ | 30問 |
| 語句の用法 | 慣用句・四字熟語・表現の正しい意味・用法を選ぶ | 30問 |
| 二語関係 | 与えられた語のペアと同じ関係を持つペアを選ぶ | 30問 |
| 文の並べ替え | バラバラな文を正しい順序に並べた選択肢を選ぶ | 25問 |
| 空欄補充 | 文章中の空欄に入る最適な語句を選ぶ | 25問 |

#### 言語：問題文（body_text）の書き方

```
語句の意味:   '次の語句の意味として最も適切なものはどれか。「〇〇（読み仮名）」'
同意語:       '次の語句と同じ意味の語句はどれか。「〇〇（読み仮名）」'
反意語:       '次の語句の反対の意味を持つ語句はどれか。「〇〇（読み仮名）」'
語句の用法:   '次の慣用句・表現の意味として最も適切なものはどれか。「〇〇」'
二語関係:     '「A：B」と同じ関係になるように、「C：＿＿」の＿＿に入る最も適切な語はどれか。'
文の並べ替え: '次のア〜エの文を意味が通るよう並べ替えたとき、正しい順序はどれか。\nア〜エの文'
空欄補充:     '次の文の（　）に入る最も適切な語句はどれか。\n文章本文'
```

#### 言語：選択肢（choices）の書き方

- 語句の意味・同意語・反意語・語句の用法：**語句または短い意味説明**
- 二語関係：「C：X」の形で、X 部分の語を選択肢に記載
- 文の並べ替え：「ア→イ→ウ→エ」のような並び順
- 空欄補充：語句のみ

### 3-2. 非言語分野（section_id=13）

| カテゴリ | 内容 | 問題数目安 |
|---|---|---|
| 割合・比率 | 割引・利益率・受験者数など割合の計算 | 25問 |
| 速さ・距離・時間 | 速度計算・相対速度・通過時間など | 25問 |
| 仕事算 | 複数人・機械の共同作業・給排水 | 25問 |
| 損益算 | 原価・定価・売価・割引・利益の計算 | 25問 |
| 集合 | ベン図・アンケート・共通部分の人数 | 25問 |
| 推論 | 順序関係・正誤判定・整数推論 | 25問 |
| 場合の数・確率 | 順列・組み合わせ・確率計算 | 25問 |
| 図表の読み取り | 表・グラフから情報を読み取る計算 | 25問 |

#### 非言語：問題文（body_text）の書き方

- 計算に必要な数値・条件をすべて problem 文中に記載する
- 「次の問いに答えなさい。」など前置きは不要。条件を直接記述する
- 答えの単位は **問題文中に明示** する（「何円か」「何人か」「何日か」「何通りか」「何%か」）
- 選択肢に数値を並べるため、4 つの選択肢が自然になるよう問題設計する

```
割合:   '定価5,000円の商品を20%引きで販売した。売値は何円か。'
速さ:   'A地点からB地点まで時速60kmで走ると2時間かかる。時速80kmで走ると何分かかるか。'
仕事算: 'AとBが協力すると6日で完成する仕事を、Aだけでやると10日かかる。Bだけでやると何日かかるか。'
損益:   '原価800円の商品に25%の利益を乗せて定価をつけた。定価は何円か。'
集合:   '100人のうち、Aを選んだ人が60人、Bを選んだ人が50人、どちらも選ばなかった人が10人いた。両方選んだ人は何人か。'
推論:   'A・B・C・D・Eの5人が一列に並んでいる。AはBの隣にいる。CはDより前にいる。BはEより後ろにいる。先頭は誰か。'
確率:   '4人の中からリーダーとサブリーダーを1人ずつ選ぶ。選び方は何通りか。'
図表:   '右の表を参照して答えなさい。\n[表の内容を問題文に埋め込む]\n〇〇は何か。'
```

#### 非言語：選択肢（choices）の書き方

- 計算結果の数値（単位なし）を選択肢に記載する
- 4 択の数値は「正解の前後・よくある計算ミス・ひっかけ値」を含める
- 単位（円・人・日・%・通り）は `choice_text` に含めず、`body_text` 側に記載する

```sql
-- 例：割合問題（正解: 4000円）
INSERT INTO choices (question_id, choice_text, image_url, is_correct) VALUES
  (currval('questions_id_seq'), '4000', NULL, true),   -- 正解
  (currval('questions_id_seq'), '3500', NULL, false),  -- 15%引きのミス
  (currval('questions_id_seq'), '4500', NULL, false),  -- 計算ミス
  (currval('questions_id_seq'), '3000', NULL, false);  -- 40%引きのミス
```

## 4. explanation（解説）の書き方

### 言語

```
語句の意味: '「〇〇」は、〜〜を意味する。'
同意語:     '「〇〇」は〜〜という意味で、「△△」と同意。'
反意語:     '「〇〇」の反意語は「△△」。〇〇は〜〜、△△は〜〜。'
語句の用法: '「〇〇」は〜〜という意味で使われる慣用表現。'
二語関係:   '「A：B」はAとBが〔関係の説明〕の関係。同様に「C：X」も同じ関係にある。'
文の並べ替え: '正しい順序は〇→〇→〇→〇。各文の接続詞・指示語に注目する。'
空欄補充:   '文脈より（　）には「〇〇」が入る。〜〜という理由による。'
```

### 非言語

- **計算式を明記する**（式 → 答え の形式）
- 1〜3行の簡潔な説明

```
割合:   '定価5000×(1−0.20) = 4000円'
速さ:   '距離=60×2=120km、時間=120÷80=1.5時間=90分'
仕事算: '全体の仕事量を1とすると、A=1/10日、B=1/6−1/10=1/15日、B単独=15日'
損益:   '定価=原価×(1+利益率) = 800×1.25 = 1000円'
集合:   '両方=60+50−(100−10) = 20人'
推論:   '条件を整理：〜〜→先頭はX'
確率:   '4×3=12通り（順列 P(4,2)）'
```

## 5. ディストラクタ（誤答）選定基準

### 言語

| カテゴリ | 誤答の選び方 |
|---|---|
| 語句の意味 | 読み方が似た語の意味／反対の意味／全く異なる意味 |
| 同意語 | 同音異義語／類似形態の別語／関連語だが意味が違う語 |
| 反意語 | 同意語を誤答に含める／無関係語を含める |
| 語句の用法 | 語句の一部から連想される誤った意味 |
| 二語関係 | 同じ語を使った別の関係のペア |
| 文の並べ替え | よくある接続詞の誤読パターン |
| 空欄補充 | 同品詞で文脈にそぐわない語 |

### 非言語

| 誤答の種類 | 具体例 |
|---|---|
| 計算ミス値 | 割合を%と小数で混同した値（×0.2 と ×20 の混同など） |
| 逆算ミス値 | 引く・足す方向を間違えた値 |
| 途中式の値 | 計算の途中段階の値（最終答えではない） |
| 単位変換ミス値 | 時間↔分、km↔m の変換を誤った値 |

4択全体で「式を立てれば確実に選べる」難易度を設定する。
「計算しなくても選べる」ほど差が大きな選択肢群は避ける。

## 8. 問題作成手順

1. **spi.sql の末尾を確認**し、言語・非言語それぞれの現在の最大 question_number を確認する。

2. **作成するカテゴリを決める**。セクション 7 のバランス表を参照し、少ないカテゴリを優先する。

3. **body_text を作成**する:
   - 言語：問題種別に応じたフォーマットを使用
   - 非言語：数値・条件を明示し、答えの単位を問題文に含める

4. **explanation を作成**する:
   - 言語：語句の意味・関係性・文脈を簡潔に説明
   - 非言語：計算式を必ず記載

5. **選択肢を作成**する:
   - 正解を 1 番目に記述
   - 誤答 3 つはカテゴリ別のディストラクタ基準に従う
   - 非言語は数値のみ（単位なし）

6. **自己検査**:
   - [ ] `question_type='choice'` になっているか（`'text'` は使わない）
   - [ ] `correct_answers` が NULL か（choice 問題のため）
   - [ ] 正解が 1 番目に配置されているか
   - [ ] body_text に答えの単位が明記されているか（非言語）
   - [ ] explanation に計算式があるか（非言語）
   - [ ] question_number が連番になっているか（欠番なし）
   - [ ] 誤答がディストラクタ基準を満たしているか
   - [ ] `image_url` が NULL か
   - [ ] section_id が 12（言語）または 13（非言語）か

## 9. よくある間違いチェックリスト

| 間違い | 正しい形 |
|---|---|
| 非言語を `question_type='text'` にする | 全問 `'choice'` で統一 |
| `correct_answers` に数値を入れる | choice 問題は常に `NULL` |
| 選択肢に単位を入れる（例: `'4000円'`） | 数値のみ（例: `'4000'`）。単位は body_text に |
| 正解を 2〜4 番目に配置する | 正解は必ず **1 番目** |
| explanation に計算式がない（非言語） | 必ず「式 → 答え」の形で記載 |
| question_number を確認せず 1 から始める | spi.sql 末尾で対象 section の最大番号を確認してから +1 する |
| 答えが 1 つに定まらない問題設計 | 問題文の条件を追加して一意の答えにする |
| 4 択の差が大きすぎる（例: 2, 20, 200, 2000） | 計算ミスから生じる「もっともらしい値」を誤答にする |
