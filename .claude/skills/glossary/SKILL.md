## 3. 問題形式の仕様

### 3-1. body_text（問題文）の書き方

**ルール：問題文に正解の用語名を含めてはならない。**
定義・特徴・条件のみで記述し、末尾は「〜はどれか。」で終わる。

**使用する出題パターン（5種類）:**

| パターン | 説明 | 例 |
|---|---|---|
| **定義型** | 用語の定義・特徴を説明して「〜はどれか。」 | `小数点の位置をあらかじめ固定して…数値表現形式はどれか。` |
| **略語型** | 正式名称を英語で説明して「略称はどれか。」または逆 | `「Binary Coded Decimal」の略称はどれか。` |
| **操作型** | 特定の操作・手順を説明して「〜操作はどれか。」 | `スタックのトップにデータを追加する操作はどれか。` |
| **穴埋め型** | 説明文の中に〔　〕で用語部分を隠す | `浮動小数点数を構成する3要素のうち，有効数字を表す〔　〕はどれか。` |
| **人物・歴史型** | 開発者・年・組織から用語/技術名を問う | `1972年にデニス・リッチーが開発した，UNIX記述にも使われた言語はどれか。` |

**body_text の長さ目安:** 1〜3文（40〜120文字程度）。冗長にしない。

### 3-2. 選択肢（choices）の書き方

| 項目 | ルール |
|---|---|
| 正解 | **必ず1番目**に記述（シャッフル可能、UI側でランダム表示） |
| 誤答数 | 3つ（計4択） |
| 誤答の選び方 | 同カテゴリ・同分野の類似用語から選ぶ（下表参照） |
| 選択肢テキスト | 用語名のみ。説明文を入れない |
| 複数正解の禁止 | 誤答が「正解にもなり得る」用語でないことを確認する |

**ディストラクタ（誤答）の選定基準:**

```
優先順位:
1. 対になる/逆の概念  例: 論理シフト ↔ 算術シフト ↔ 循環シフト
2. 同じ操作・役割をもつ別の方式  例: スタック ↔ キュー ↔ デキュー
3. 上位/下位/隣接概念  例: 単精度浮動小数点数 ↔ 倍精度 ↔ 半精度
4. 試験で混同されやすいペア  例: 相関分析 ↔ 回帰分析 ↔ 因子分析
```

4択全体で「知識があれば確実に排除できる」レベルの難易度を設定する。
全選択肢が正解候補に見える「罠問題」は避ける。

### 3-3. explanation（解説）の書き方

**必須要素（この順で記述）:**

1. **正解の根拠**: 用語の正式定義・核心的特徴（1文）
2. **補足知識**: 規格番号・計算量・数値情報・活用場面（1〜2文）
3. **誤答との区別**（混同されやすい場合のみ）: 主要誤答との違いを1文で

**explanation の長さ目安:** 2〜4文（60〜200文字程度）。

## 5. 問題作成手順

1. **対象セクションの最大 question_number を確認**する。
   ```sql
   -- glossary.sql の末尾で当該 section_id の最大番号を確認
   ```

2. **サブカテゴリのバランスを確認**し、問題が少ないトピックを選ぶ（セクション 4 参照）。

3. **body_text を作成**する:
   - 正解用語名を文中に含めない
   - 末尾は「〜はどれか。」
   - セクション 3-1 のパターンのいずれかに合致させる

4. **選択肢を作成**する:
   - 正解を1番目に記述
   - 誤答3つはセクション 3-2 のディストラクタ選定基準に従う

5. **explanation を作成**する:
   - 正解の根拠 → 補足知識 → 誤答との区別（必要なら）

6. **自己検査**:
   - [ ] body_text に正解用語名が含まれていないか
   - [ ] 末尾が「〜はどれか。」になっているか
   - [ ] 正解が1番目に配置されているか
   - [ ] 誤答4択に「複数正解」がないか（他の選択肢も正解にならないか）
   - [ ] ディストラクタが同カテゴリの用語か
   - [ ] explanation が正解の根拠を含んでいるか
   - [ ] question_number が連番になっているか（欠番なし）

## 6. SQL テンプレート

```sql
-- section_id=XX, Q{N}: {用語名}
DO $$
DECLARE
  qid integer;
BEGIN

  INSERT INTO questions (section_id, question_number, question_type, max_selections, score, body_text, explanation)
  VALUES (XX, N, 'choice', 1, 1,
    '定義・特徴の説明文。末尾は「〜はどれか。」',
    '正解の根拠（正式定義・核心的特徴）。補足知識（規格・計算量・活用場面）。誤答との区別（混同例がある場合）。'
  ) RETURNING id INTO qid;
  INSERT INTO choices (question_id, choice_text, is_correct) VALUES
    (qid, '正解用語名',    true),
    (qid, '誤答用語名A',   false),
    (qid, '誤答用語名B',   false),
    (qid, '誤答用語名C',   false);

END $$;
```

**複数問まとめて1ブロックに記述する場合:**

```sql
DO $$
DECLARE
  qid integer;
BEGIN

  -- Q{N}: {用語名1}
  INSERT INTO questions (section_id, question_number, ...) VALUES (...) RETURNING id INTO qid;
  INSERT INTO choices (...) VALUES (...);

  -- Q{N+1}: {用語名2}
  INSERT INTO questions (section_id, question_number, ...) VALUES (...) RETURNING id INTO qid;
  INSERT INTO choices (...) VALUES (...);

END $$;
```

## 7. よくある間違いチェックリスト

| 間違い | 正しい形 |
|---|---|
| `body_text` に正解用語を書く | 用語名を伏せて定義・特徴のみを記述する |
| 末尾が「〜である。」「〜とは何か。」 | 必ず「〜はどれか。」で終わる |
| 正解を2〜4番目に配置 | 正解は必ず **1番目** |
| 誤答に正解と無関係な別分野の用語を使う | 同カテゴリ・類似概念から選ぶ |
| `explanation` が「正解は〇〇です」のみ | 根拠・補足知識・活用場面を含める |
| 連番を飛ばす（Q5 の次が Q7 など） | 必ず連続した番号を使う |
| DO $$ ブロックを使わず currval を使う | glossary.sql は DO $$ パターン（パターン A）を使う |
| `RETURNING id INTO qid` を省略する | 必須（choices の question_id に使用） |
